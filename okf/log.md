# Update Log

## 2026-06-23
* **Fix**: Replaced the "reserve the last column" workaround with the documented ncurses mechanism. Per `curs_addch(3x)`, with `scrollok()` FALSE a write that advances the cursor off the bottom-right margin returns `ERR` instead of scrolling the screen — so the frame and both bars now span the full `COLS` (the right border reaches the screen edge, no blank column) and the top menu bar stays pinned. The earlier "status bar stops one column short" note is superseded.
* **Fix**: Added 1-column padding inside both panes so body/tree text never butts against the frame borders (the borders stay clean and the content has breathing room). Body wraps to `cols-split-4`; verified long lines wrap rather than truncate.
* **Fix**: Corrected the OKFI logo art (rows were uneven widths → garbled letters; now verified 28 columns each, literal UTF-8). Made rendering auto-margin-safe: the last terminal column is now never written (bars stop at `COLS-1`; both panes lay out within `COLS-1`), so a long line can no longer trigger a one-row scroll that ate the top menu bar.
* **Update**: Renamed the project **fokf → okfi** (Open Knowledge Format Interface) — binary, config dir (`~/.config/okfi/`), source `okfi.c`, temp files, and all branding.
* **Update**: v3 visual overhaul — a gradient **OKFI ANSI logo** (cyan→magenta) heads the picker; real **dropdown menus** (`File · Edit · View · Settings · Help`, `F9`/`\`); four readable **themes** (`dark`/`light`/`bbs`/`mono`) replacing the washed-out default; a connected single-line **frame** with the **focused pane outlined/highlighted**; the open **bundle name** in the menu bar and **path** in the status bar; `Tab` folds the current group and `Shift+Tab` folds all; `Esc` returns to the picker; a hard top/bottom-bar constraint with `KEY_RESIZE` handling. Revised `tui-viewer.md`, `config.md`.

## 2026-06-22
* **Creation**: Added a top-level `README.md` (GitHub-flavored: shields, alerts, collapsible sections, a mermaid discovery diagram, key/flag/config tables) linking into this catalog. Published the repo at github.com/theesfeld/okfi.
* **Fix**: Audit follow-ups — `build_tree_view` group arrays are now dynamic (sized to `nconcepts`), removing the 128-distinct-type cap that could silently drop a concept (accuracy floor); selecting the `priority` group order in settings now prompts for the type list, so the mode is fully UI-configurable rather than config-file-only.
* **Update**: Collapse state now **persists across runs** — folds are stored in the config as `fold = bundle⇥group` lines keyed by bundle path (the single source of truth; the session-only array was deleted). Added `*` to **collapse/expand all** groups, and **configurable group order** (`group_order = type|count|priority` + `group_priority` list), cyclable in settings. Verified all three end-to-end.
* **Fix**: Hardened the collapse feature per audit — widened `collapsed_groups` to match `VRow.label` (a ≥64-char type name would otherwise never match in `is_collapsed`), and replaced the fragile pointer-identity reset with an explicit `load_bundle(dir, reset_collapse)` parameter (folds now reliably survive an in-bundle edit/new, reset only on bundle switch).
* **Update**: Tree groups are now **collapsible** (`Space`/`Enter` on a header, `▾`/`▸` markers) and show a **per-type count**; a selected header shows a folder summary in the content pane. Both panes sit in a **frame** whose focused pane is outlined/brightened with a highlighted `TREE`/`CONTENT` title. Collapse state is per-bundle session state.
* **Update**: Browser polish — left pane is now a **type-grouped tree** (headers per `type`, reserved files in their own group); added **pane focus** (`l`/`h` or arrows) so the content pane scrolls independently; fixed a screen-scroll bug where filling the bottom-right cell pushed the top bar off (status bar now stops one column short, `scrollok` pinned off). Editor discard is now a **single keypress**; `prompt_line` accepts **UTF-8** input; an `editor = internal|system` setting chooses okfi's editor or `$EDITOR`; discovery reads only the `index.md` prefix and the picker rescans only when roots change. Revised `tui-viewer.md`, `config.md`.
* **Fix**: Closed three v2 floor bugs from the audit — editor save failure no longer silently discards the buffer (keeps it, shows the error); open→save round trip is byte-identical (dropped the trailing-newline split artifact); `collect()` is depth-capped and skips symlinks (no stack overflow on symlink loops). All verified.
* **Update**: v2 — okfi is now a multi-bundle browser, editor, and scaffolder, not just a single-bundle viewer. Added: bundle **discovery** over XDG-configured search roots + a project→project **picker**; an in-TUI **editor** (`e`; codepoint-aware, atomic temp+rename save, `^O` save / `^X`-`Esc` cancel); **create** flows (`n` concept, `N` bundle; CLI `--new-bundle`/`--new-concept`) that seed conformant skeletons and prepend dated `log.md` entries; a **settings** screen and XDG **config** (`root`, `theme` default/bbs/mono, per-role `color.*`; unknown keys preserved); `?` **help** overlays; a BBS skin (menu bar, box-drawing banner, themed palette). New flags `--root`, `--help`. New concept docs: `discovery.md`, `editor.md`, `create.md`, `config.md`; revised `tui-viewer.md`, `build.md`, `index.md`. Fixed an overlapping-`snprintf` segfault in `load_bundle` and `^S` being eaten by tty flow control.
* **Update**: Inline spans now nest — `style_run`/`latex_run` recurse into bold/italic/link spans (layering attributes / wrapping LaTeX) so e.g. a code span inside bold renders, in both the TUI and the PDF export. Code spans stay literal; recursion is depth-capped (`INLINE_MAX_DEPTH`) against pathological nesting. Self-checked both paths.
* **Update**: Added soft-line-break reflow — a shared `read_block` joins lazy-continuation lines into one logical paragraph (bullets/blockquotes absorb their continuations) before inline styling, so emphasis spanning a soft break in the source renders in both the TUI and the PDF export. Self-checked.
* **Update**: Inline code now follows CommonMark backtick-run spans (`` `code` `` embeds single backticks) via a shared `code_span` helper; LaTeX export escapes backticks (`\textasciigrave`) so triple backticks no longer render as quote ligatures.
* **Update**: Applied PDF-exporter audit fixes — frontmatter URL values escaped unless a clean `\url` (closing an escaping-floor hole), `copy_file` now detects short-write/`fclose` errors and unlinks partial output, temp dir removed on the `fopen`-fail path. Switched fenced code from `verbatim` to `listings`/`lstlisting` (tolerates `\end{verbatim}` content; residual edge only on literal `\end{lstlisting}`).

Reuses the shared block classifiers; headings→Section/`\milpar`, tables→booktabs (alignment honored), code→verbatim, links→`\href`, frontmatter→booktabs table. LaTeX escaping (with `\texttt` hyphen-break) self-checked via `open_memstream`. Extracted `classify_line`/`table_block` so the TUI and PDF paths share one classifier; `pdf-export.md` updated planned→built.
